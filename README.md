# MCP-Agent-Server: AI 협업 개발 환경

`mcp-agent-server`는 Claude, Copilot, Gemini, Perplexity 등 여러 AI 모델을 개발 에이전트로 활용하여, 복잡한 소프트웨어 개발 과제를 협업을 통해 해결하는 것을 목표로 하는 프로젝트입니다.

## 개발 로드맵 (Development Roadmap)

프로젝트는 총 4단계의 점진적인 개발 계획을 따릅니다.

---

### **Phase 1: 핵심 기반 구축 (Core Foundation)**

시스템의 기본 골격을 만들고, 단일 AI 에이전트를 연동하여 최소 기능의 워크플로우를 구현합니다.

*   **1. 중앙 오케스트레이터 서버 구축 (Orchestrator Server)**
    *   `Node.js (Express or NestJS)` 또는 `Python (FastAPI)` 기반의 백엔드 서버를 설정합니다.
    *   개발 작업을 정의하고, 상태를 추적하며, 결과를 반환하는 핵심 API 엔드포인트를 설계합니다.

*   **2. 첫 번째 AI 에이전트 연동 (First Agent Integration)**
    *   주력으로 사용할 AI 모델 (예: Gemini Pro)의 API와 연동하는 `어댑터(Adapter)`를 개발합니다.
    *   어댑터는 `(인증) -> (요청 형식 표준화) -> (API 호출) -> (응답 파싱)`의 역할을 수행합니다.

*   **3. 파일 시스템 제어 (File System Control)**
    *   AI 에이전트가 지정된 워크스페이스 내의 파일을 읽고, 수정하며, 생성할 수 있는 기능을 구현합니다.

---

### **Phase 2: 멀티 에이전트 확장 (Multi-Agent Expansion)**

여러 종류의 AI 에이전트를 추가하고, 작업 성격에 따라 최적의 에이전트를 선택하는 기능을 구현합니다.

*   **1. AI 에이전트 어댑터 추가 (Additional Agent Adapters)**
    *   `Claude`, `Perplexity` 등 다른 AI 모델의 API 어댑터를 추가로 구현합니다.
    *   *Note: Copilot은 직접 API가 제한적이므로, 유사한 OpenAI 모델로 대체하거나 다른 연동 방식을 고려합니다.*

*   **2. 에이전트 라우팅 로직 (Agent Routing)**
    *   작업의 유형(예: `코드 생성`, `디버깅`, `문서 작성`)에 따라 가장 적합한 AI 에이전트를 선택하여 작업을 할당하는 `라우터(Router)`를 개발합니다.

*   **3. 공유 워크스페이스 (Shared Workspace)**
    *   모든 에이전트가 동일한 파일 상태를 공유하며 충돌 없이 작업할 수 있는 `격리된 워크스페이스` 관리 기능을 구현합니다.

---

### **Phase 3: 사용자 인터페이스 및 상호작용 (UI & Interaction)**

사용자가 AI의 개발 과정을 모니터링하고, 결과를 검토하며, 직접 제어할 수 있는 인터페이스를 제공합니다.

*   **1. 사용자 인터페이스 개발 (UI Development)**
    *   **초기:** 빠르게 개발할 수 있는 `CLI (명령줄 인터페이스)`를 구현합니다.
    *   **장기:** `웹 기반 대시보드 (React/Vue)`를 구축하여 사용성과 시각화를 개선합니다.

*   **2. 인터랙티브 워크플로우 (Interactive Workflow)**
    *   사용자가 `개발 목표`를 입력하면, 오케스트레이터가 이를 해석하여 AI 에이전트에게 작업을 할당하고, 진행 상황을 UI에 실시간으로 표시합니다.
    *   AI가 제안한 코드 변경사항(`diff`)을 UI에서 명확하게 확인하고, `승인(Apply)` 또는 `반려(Reject)`할 수 있는 기능을 구현합니다.

---

### **Phase 4: 개발 자동화 및 고도화 (Automation & Advanced Features)**

복잡한 개발 과제를 AI가 스스로 하위 작업으로 나누고, 에이전트 간의 협업을 통해 해결하도록 시스템을 고도화합니다.

*   **1. 자동 작업 분해 (Automatic Task Decomposition)**
    *   "로그인 기능 개발"과 같은 복잡한 요청을 오케스트레이터가 `(API 설계) -> (코드 작성) -> (테스트 작성)` 등 구체적인 하위 작업으로 자동 분해하는 엔진을 개발합니다.

*   **2. 에이전트간 협업 워크플로우 (Inter-Agent Collaboration)**
    *   `코딩 에이전트`가 작성한 코드를 `리뷰 에이전트`가 검토하고, `테스트 에이전트`가 실행하는 등, 역할이 분담된 에이전트들이 협력하는 워크플로우를 구현합니다.

*   **3. 자동 피드백 루프 (Automated Feedback Loop)**
    *   `테스트 실패`나 `린트(Lint) 오류`가 발생하면, 해당 결과를 즉시 담당 AI 에이전트에게 피드백하여 자동으로 코드를 수정하도록 하는 기능을 구현합니다.
